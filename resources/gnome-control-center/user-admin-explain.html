관리 권한은 어떻게 사용되나요? 
시스템이 올바르게 작동하는 데 필요한 파일이 잘못 변경되면 시스템에 문제가 발생하기 때문에 관리자 권한을 사용하여 무단으로 변경되는 것을 보호해야 합니다. 또한 일부 애플리케이션은 시스템에 필요한 필수 요소를 변경하기 때문에 보호되어야 합니다.

시스템을 보호할 수 있는 방법은 관리자 권한이 있는 사용자만 파일을 변경하거나 애플리케이션을 사용할 수 있도록 하는 것입니다. 일반적으로는 시스템 파일을 변경하거나 중요 애플리케이션을 사용할 필요가 없으므로 관리 권한이 없어도 됩니다. 

하지만 이런 애플리케이션을 사용해야 하는 경우 변경 작업을 수행할 수 있도록 일시적으로 관리자 권한을 부여 받아야 합니다. 애플리케이션에 관리자 권한이 필요한 경우 암호 입력을 요청합니다. 예를 들어 일부 새 소프트웨어를 설치하려는 경우 소프트웨어 설치 관리자(패키지 관리자)가 새 응용 프로그램을 시스템에 추가할 수 있도록 관리자 암호를 요청합니다. 작업이 완료되면 관리자 권한이 다시 없어집니다.

관리자 권한은 사용자 계정과 연동되며 관리자 권한이 있는 사용자는 표준 사용자가 아닌 동안 이러한 권한을 가질 수 있습니다. 관리자 권한이 없으면 소프트웨어를 설치할 수 없습니다. 일부 사용자 계정(예: "root" 계정)에는 영구적인 관리 권한이 있습니다. 의도하지 않은 항목(예: 필요한 시스템 파일 삭제)을 실수로 변경할 수 있으므로 관리자 권한을 상시로 소유해서는 안 됩니다.

다시 말해 관리자 권한은 필요한 경우 시스템의 중요한 부분을 변경할 수 있도록 허용하지만 실수로 변경하는 것은 방지합니다.
'슈퍼 사용자'란 무엇인가요?

관리자 권한을 가진 사용자를 '슈퍼 사용자'라고도 합니다. 슈퍼 사용자는 일반 사용자보다 더 많은 권한을 가지고 있으며 su 및 sudo와 같은 명령어를 사용합니다. 이는 일시적으로 슈퍼 사용자 권한을 부여하는 데 사용하는 명령어입니다.


관리 권한이 유용한 이유는 무엇입니까? 
시스템 변경 전 사용자에게 관리 권한 요구하는 것은 시스템이 의도적으로 또는 의도하지 않게 손상되는 것을 방지하는 데 도움이 됩니다.

관리자 권한을 가진 사용자가 중요한 파일을 실수로 변경하거나 중요한 파일을 실수로 변경하는 응용 프로그램을 실행할 수 있습니다. 만약 관리자 권한을 필요할 때 일시적으로 부여한다면 이러한 실수를 사전에 방지할 수 있습니다.

신뢰할 수 있는 특정 사용자에게만 관리자 권한을 부여해야 합니다. 이렇게 하면 다른 사용자가 컴퓨터를 무단으로 사용하거나 필요한 애플리케이션을 제거하거나 불법 애플리케이션을 설치하거나 중요한 파일을 변경하는 작업을 수행할 수 없어 강력한 보안을 유지할 수 있습니다.
